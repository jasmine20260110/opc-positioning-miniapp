# OPC定位神器：公开版功能说明

## 产品目标

帮助处于转型、副业探索或个人定位阶段的用户，把“我可能适合做什么”转化为“我现在最值得验证什么”。

## 目标用户

- 有多种兴趣或能力，但难以确定优先方向的人；
- 想转型或尝试副业，希望先进行低成本验证的人；
- 已经积累了一些经历，但难以从中提炼稳定优势的人。

## MVP 输入与输出

### 用户输入

- 过往投入经历；
- 容易进入专注状态的活动；
- 已有能力与成果；
- 当前时间、身份和资源约束。

### 产品输出

- 可回溯到用户回答的个人证据；
- 3 条候选路线；
- 每条路线的公开层面说明；
- 1 条用户选择的优先路线；
- 7 天最小验证行动。

## 设计原则

1. **证据优先**：结论应尽量对应用户提供的经历；
2. **区分事实与推测**：证据不足时明确提示，而不是假装确定；
3. **低后悔验证**：先用小行动验证，不鼓励直接做重大人生决定；
4. **用户保留决定权**：系统提供候选路线，最终方向由用户选择。

## 公开与隐私边界

- 本仓库公开小程序源代码、模型 Prompt、数据契约、测试脚本和演示材料，供项目评审查看；
- 真实 API Key、云环境密钥、`.env` 和 `project.private.config.json` 不公开；
- 真实用户回答、本地反馈记录、客户信息和其他个人数据不进入 Git；
- 仓库内的回答、数字、路线和截图均为虚构演示内容；
- 内部成本数据和未定商业方案不作为已验证事实公开宣称。

## 项目阶段

当前为 MVP Demo，重点验证用户是否能理解候选路线、是否愿意选择其中一条，以及 7 天行动能否真正开始执行。
