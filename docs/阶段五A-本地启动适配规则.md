# 阶段五A：本地启动适配规则

## 范围

阶段五A不调用AI，只比较“路线启动要求”和“用户当前启动条件”，并生成固定7天验证计划。

## 内部状态

| 内部分数 | 页面状态 | 含义 |
|---|---|---|
| 2 | 当前满足 | 用户条件稳定覆盖路线最低要求 |
| 1 | 可以补足 | 用户区间部分覆盖，或有明确补足方式 |
| 0 | 明显冲突 | 当前条件达不到路线最低要求 |
| null | 待验证 | 路线要求或用户条件缺少可比较信息 |

内部数值只参与程序计算，页面不展示。

## 四项比较规则

1. 首笔变现：同时比较最晚收入期限和最低有意义金额，取更弱的一项。
2. 稳定投入时间：用户时间下限达到要求为“当前满足”；只有上限能覆盖为“可以补足”。
3. 首批用户资源：可直接联系人覆盖要求为“当前满足”；人数不足但已有联系人和补足方式为“可以补足”；0人且无法补足为“明显冲突”。
4. 验证预算：预算覆盖最低要求为“当前满足”；预算不足但愿意花钱或合作为“可以补足”；否则为“明显冲突”。

## 路线总结果

- 任一维度为0：暂缓；
- 没有0但存在待验证：待验证；
- 没有0和待验证但存在1：补足后验证；
- 四项全部为2：优先验证。

最大缺口依次取第一个“明显冲突 → 可以补足 → 待验证”的维度。四项全部满足时，仍提示市场需求需要真实验证。

## 固定7天计划

固定计划正好包含Day 1—Day 7，每天只有一项最小行动，并包含预计用时、完成证据和降级方案。Day 6固定测试付费信号，避免计划只有学习任务。

阶段五B已经接入第4次AI调用；真实分析优先生成个性化计划，本地固定计划继续作为演示模式和AI失败时的降级方案。
