# 阶段四：阿里百炼配置与验证

## 当前实现

- 云函数：`cloudfunctions/opcApi`；
- 默认模型：`deepseek-v4-flash`；
- 三次调用：证据提取、3条路线、市场机会和启动要求；
- Key只从云函数环境变量读取；
- DeepSeek V4 Flash不使用原生`response_format`，改用JSON提示、解析、Schema校验和两次自动重试；
- 真实分析失败时，用户可手动重试；累计3次仍失败后停止请求，保留全部答案并返回首页。

## 必须先处理

聊天中出现过的旧API Key应立即在百炼控制台禁用并重新生成。不要再次把新Key发送到聊天，也不要写进代码、`.env.example`或截图。

## 云函数环境变量

在微信云开发控制台的`opcApi`云函数配置中添加：

| 变量 | 必填 | 值 |
|---|---|---|
| `DASHSCOPE_API_KEY` | 是 | 新生成的百炼Key |
| `DASHSCOPE_BASE_URL` | 推荐 | `https://你的WorkspaceId.cn-beijing.maas.aliyuncs.com/compatible-mode/v1` |
| `DASHSCOPE_MODEL` | 否 | 默认已经是`deepseek-v4-flash` |
| `DASHSCOPE_TIMEOUT_MS` | 否 | 默认`25000`，不要超过`25000` |
| `DASHSCOPE_MAX_COMPLETION_TOKENS` | 否 | 默认`6000` |

不要在小程序端配置这些变量。

## 小白操作步骤

1. 在阿里百炼控制台禁用已暴露的旧Key，创建新Key。
   - 成功标志：旧Key显示已禁用，新Key仅保存在你的密码管理器或云函数环境变量中。
2. 在百炼业务空间确认已开通`deepseek-v4-flash`，复制业务空间ID。
   - 成功标志：模型API页面显示模型ID`deepseek-v4-flash`和北京地域调用地址。
3. 在微信开发者工具中选择当前云环境，上传并部署`opcApi`，勾选云端安装依赖。
   - 成功标志：云函数列表出现`opcApi`，部署状态成功。
4. 在`opcApi`配置页开启公网访问，并把函数执行超时设为60秒。
   - 成功标志：函数可以访问公网，配置页的执行超时不低于60秒。
5. 在`opcApi`配置页添加上表环境变量并保存，然后重新部署或重启云函数实例。
   - 成功标志：调用`health`操作返回`configured: true`，且不会返回Key本身。
6. 用一组不含隐私和公司机密的20题答案走完整流程。
   - 成功标志：Loading页三个阶段依次完成，结果正好包含3条路线，每条正好5项市场机会。

## 隐私与日志

- 20题会经微信云函数发送给阿里百炼处理；
- 云函数不把完整答案写入日志；
- 结果保存在小程序本机，不写云数据库；
- 内部Demo也不要输入身份证号、客户名单、公司机密等敏感信息。
