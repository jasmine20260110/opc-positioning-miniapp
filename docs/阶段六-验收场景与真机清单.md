# 阶段六：验收场景与真机清单

## 验收边界

产品：OPC定位神器内部MVP。

用户角色：被加入微信小程序体验成员的内部测试者。

本阶段验证P0主链路、故障降级、本地恢复、安全边界和体验版可用性；不验证登录、云数据库、打卡、提醒或跨设备同步。

## 场景1：正常真实AI主链路

**目标：** 验证真实回答可以生成完整结果。

**起始条件：** `opcApi`已部署，Key和模型已配置，云函数超时60秒，公网访问开启。

**步骤与预期：**

1. 从首页开始并完成20题 → 每题即时保存在本机；
2. 提交 → Loading依次完成证据、路线、市场3个阶段；
3. 查看报告 → 证据引用原文，路线正好3条，每条市场机会正好5项；
4. 确认4项启动条件 → 每条路线展示4项适配状态，不显示内部精确分数；
5. 选择路线 → DeepSeek生成正好7天计划，至少一天验证市场或付费；
6. 点击“开始 Day 1” → 本机记录开始状态，按钮变为“Day 1 已开始”。

## 场景2：退出、返回和恢复

**目标：** 验证关键状态不会因退出页面丢失。

**步骤与预期：**

1. 在Q3输入内容但不点击下一题，返回首页再继续 → 恢复Q3和草稿；
2. 修改Q20 → 每周时间预填同步更新；
3. 生成启动适配后返回修改条件 → 可以重新计算；
4. 生成计划后重新选择路线 → 返回启动适配结果；
5. 开始Day 1后重新打开计划 → 仍显示已开始。

## 场景3：AI失败降级

**目标：** 验证网络、超时或结构错误不会形成空白页。

**步骤与预期：**

1. 单次AI调用失败 → 系统自动重试2次；用户手动重试累计3次仍失败后停止请求，可返回首页且保留全部答案；
2. 第4次计划AI失败 → 自动生成本地固定7天计划并显示来源说明；
3. 降级计划 → 仍包含7天、行动、用时、证据、降级方案和付费验证。

## 场景4：三组测试数据

1. A：能力证据较强，市场证据较弱；
2. B：已有付费信号，但时间和资源不足；
3. C：能力方向较多，收入期限和用户资源受限。

每组检查20题、原文证据、3条路线、每条5项市场机会、固定规则适配和7天AI计划。

## 场景5：安全与产品边界

**预期：**

- 小程序代码中不存在真实API Key；
- 云函数日志不记录完整答案；
- 页面不展示星级、精确适配分数或打卡入口；
- 隐私提示明确说明答案会发送至微信云函数和阿里百炼；
- AI失败时不展示供应商原始技术错误。

## 真机验收清单

1. 用非开发者体验成员微信扫码打开体验版；
2. 分别检查iPhone和一台安卓机的文字溢出、按钮点击、横向路线卡滑动；
3. 完成一次真实20题主链路；
4. 将小程序切到后台再返回，确认当前题和答案保留；
5. 断网触发错误，确认可以重试或返回首页；累计3次手动失败后不再发起请求；
6. 点击示例数据按钮，确认进入问答页且已有20条预填答案；
7. 选择路线并开始Day 1，重新进入后确认状态恢复。

成功标志：非开发者体验成员可以独立完成主链路，无空白页、无阻断错误。
