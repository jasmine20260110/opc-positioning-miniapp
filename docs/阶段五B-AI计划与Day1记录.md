# 阶段五B：AI计划与Day 1记录

## 已实现范围

- 用户查看启动适配后选择一条路线；
- 真实分析结果调用`opcApi.generatePlan`生成个性化7天计划；
- 默认模型为`deepseek-v4-flash`；
- 计划必须正好包含Day 1—Day 7；
- 每天包含唯一目标、最小行动、预计用时、完成证据和降级方案；
- 至少一天包含市场或付费验证；
- AI超时、网络失败或结构错误时自动使用本地固定计划；
- 点击“开始 Day 1”后，在本机记录开始状态、时间和路线，并显示成功弹窗；
- 重新打开相同路线的计划页时恢复“Day 1 已开始”状态。

## 数据边界

- 第4次AI调用只发送用户已选择的单条路线和4项启动条件，不发送另外两条路线；
- API Key仍只保存在云函数环境变量；
- 计划和Day 1状态只保存在小程序本机，不写云数据库；
- 不包含打卡页、提醒、Day 7总结或跨设备同步。

## 计划来源

| `source` | 含义 |
|---|---|
| `ai_generated` | 阿里百炼真实生成的个性化计划 |
| `fixed_local_fallback` | 演示模式或AI失败时使用的本地固定计划 |

计划页会明确展示当前来源，但两种来源都必须满足7天完整结构。

## 验证结果

- 云函数单元测试：通过；
- 页面逻辑测试：AI成功、失败降级、Day 1记录和状态恢复均通过；
- 真实百炼调用：`deepseek-v4-flash`一次返回成功，7天结构完整并包含付费验证；
- 云函数`opcApi`已部署至当前云环境。
